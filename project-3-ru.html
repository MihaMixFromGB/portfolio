Обзор проекта

News Blog - это мой первый Fullstack проект, который я создал с помощью Nx - интеллектуальной и расширяемой системы сборки проектов, поддерживающей стратегию разработки приложений на основе монорерозиториев.

Backend

Api - backend часть проекта, которая представляет собой Restful веб-сервис, созданный с использованием прогрессивного Node.js фреймворка NestJS. Веб-сервис позволяет авторизованным пользователям управлять ресурсами, используя HTTP методы (GET/POST/PATCH/DELETE). Для получения дополнительных сведений нажмите сюда. Описание Rest API сделано с использованием библиотеки swagger.io.
Были разработаны следующие типы ресурсов (сущностей): новость, комментарий, пользователь, категория. Для хранения данных была выбрана БД PostgreSQL. Для сопоставления объектов бизнес-логики и данных (сущностей), хранимых в БД, использовались паттерн Active Record и библиотека TypeORM.
Для аутентификации пользователей использовалась библиотека passport.js (модуль @nestjs/passport). По существу, стратегия работы passport.js заключается в выполнении следующих этапов:
1) Аутентификация пользователя посредством верификации его данных (email / пароль) (local.strategy).
2) Управление состоянием пользоваля, прошедшего аутентификацию (предоставление портативного токена (JWT) посредством установки cookie с атрибутом Max-Age равным 60 секунд) (jwt.strategy).
3) Прикрепление информации (userId, email, role) о пользователе, прошедшем аутентификацию, к объекту Request для дальнейшего использования в обработчиках маршрута (jwt.strategy).
В Socket Gateway реализована логика выполнения широковещательной рассылки части клиентов с использованием механизма room от socket.io (room определяется по id новости). Внутри Comments Module для уведомления Socket Gateway о создании, редактировании или удалении комментария (логика реализована в Service) использован паттерн Event Emitter (@nestjs/event-emitter модуль).

Frontend

Web - frontend часть проекта. Здесь разработано несколько простых компонентов: форма для регистрации пользователя, страница с перечнем новостей и страница с детальной информацией о новости, где Вы также можете оставить свой комментарий. Если другой пользователь создаст, отредактирует или удалит комментарий, то Вы практически сразу будете уведомлены, получив всплывающее уведомление в правом нижнем углу экрана. SPA приложение построено с использованием следующего набора Javascript библиотек: React, React Router, TailwindCSS, Socket.io.
Разработаны два пользовательских контекста для передачи данных об авторизации и socket-сервиса по дереву React компонентов. Эти пользовательские контексты добавлены в два хука, которые предоставляют доступ к контексту внутри каждого React компонента посредством вызова соответвующего хука. Хук useAuth содержит контекст с данными авторизации: user (если не выполнен вход, то значение null), loading (true/false для установки прелоадеров при отправке запросов), error (сообщение об ошибке в объекте Response), login (отправка запроса на подключение к веб-сервису и входа в приложение), signup (отправка запроса на регистрацию нового пользователя), logout (отправка запроса на отключение от веб-сервиса и выхода из приложения). Хук useSocket содержит контекст только с одним экземпляром socket-сервиса, который позволит любому React компоненту выполнить обмен сообщениями с backend частью проекта, используя средства библиотеки socket.io.
Также в разработан компонент Protected Route, который предоставляет доступ к SPA приложению только авторизованным пользователям.
Также добавлены модульные тесты для React компонентов с использованием библиотек Jest + React Testing Library.
