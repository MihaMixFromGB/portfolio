Project Overview

News Blog is my first Fullstack project that I've built with Nx is a smart, extensinble build system with monorepo support.

Backend

The backend project's name is api. It is a RESTful Web Service based on a progressive Node.js framework NestJS that let authorized users to manipulate resources using HTTP GET/POST/PATCH/DELETE methods. For more information about the Rest API, click here. I've used swagger.io for creating Rest API description.
Some types of resources (entities) have been developed. These are a news, a comment, a user, a category. A PostgreSQL Database has been choosen for storing data. TypeORM library has been used for an object-relational mapping of our resources with the Active Record pattern.
Passport library (@nestjs/passport module) used for users authentication. At a high level, Passport executes a series of steps to:
1) Authenticate a user by verifying their "credentials" (email/password) (local.strategy).
2) Manage authenticated state (by issuing a portable token (JWT), set cookie with Max-Age attribute is 60 sec) (jwt.strategy).
3) Attach information about the authenticated user to the Request object for further use in route handlers (userId, email, role) (jwt.strategy).
The Event Emitter pattern (@nestjs/event-emitter module) used to notify the Socket Gateway of creating, editing or removing comment in the Service inside the Comments Module. The Socket Gateway implements a logic to broadcast events to a subset of clients (that join to room by newsId).

Frontend

The frontend project's name is web. There are a few simple features on the Frontend side. These are a login form, a page with a list of news and a page with news details where you can also post a comment. If other user's created, edited or removed a comment you'll be notificated immediately with a pop-up notification that will appears on the bottom right corner of the screen. The SPA is built with a collection of Javascript libraries: React, React Router, TailwindCSS, Socket.io.
Two custom contexts are designed to pass authorization and socket data through the component tree. These custom contexts are wrapped in two hooks that get access to the context inside every React component by calling the hook. The useAuth hook contains the context with auth data: user (if not login user is null), loading (true/false for setting prealoder by sending requests), error (an error's message of a response), login (send a login request), signup (send a signup request), logout (send a logout request). The useSocket hook contains the context with only one socket instance that let any React component to exchange messages with the backend using socket.io utilities.
The Protected Route component is also designed only authorized users get access to the SPA.
Unit tests of React components are also added using Jest and React Testing Library.
